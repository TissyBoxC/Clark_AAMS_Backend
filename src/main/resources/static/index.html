Capabilities

为课表导入和维护准备的核心能力

校

学校配置

后端按学校拆分能力，当前已内置金陵科技学院 JIT 配置，并保留继续扩展的结构。

导

教务导入

通过登录会话提交教务数据，服务端解析课程信息，返回客户端统一的课程模型。

版

版本检查

客户端可向后端查询更新策略、发布说明与下载来源，支持可选更新和强制更新。

管

管理入口

后端自带静态管理页，用于维护学校脚手架与版本配置，便于后续适配更多学校。

Architecture

移动端保留本地体验，服务端专注公共能力

Clark AAMS 的客户端使用 Flutter 与 Material 3 构建课程表体验， 课程与设置保存在本地。Spring Boot 后端则承担学校列表、教务解析、 健康检查与版本策略，减少客户端中与学校教务系统强绑定的逻辑。

Client Flutter / Dart

课程表、设置、导入流程、本地存储

Backend Spring Boot 4 / Java 21

学校配置、教务解析、版本检查、管理页面

Data 统一课程 JSON

课程名称、周次、节次、地点、教师、原始时间

API
