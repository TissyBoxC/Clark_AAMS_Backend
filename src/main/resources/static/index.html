Workflow

把复杂的教务接入压缩成几步

课程表应用不应该把每个学校的教务逻辑都塞进客户端。 Clark AAMS 把适配、识别、版本策略和接口文档集中到服务端，让客户端只关心稳定的数据结构。

01

学校能力配置

按学校维护登录、解析和导入能力，客户端通过统一接口读取可用能力。

02

课表数据转换

教务 HTML、截图识别结果和人工导入都收敛成同一份课程 JSON。

03

客户端稳定消费

Flutter 端只负责展示、编辑和本地存储，不被教务系统变化牵着走。

Architecture

移动端保留本地体验，服务端专注公共能力

Clark AAMS 的客户端使用 Flutter 构建课表体验，课程与设置保存在本地。 Spring Boot 后端承担学校列表、教务解析、AI 识别、健康检查与版本策略， 减少客户端与学校教务系统的强绑定逻辑。

ClientFlutter / Dart
BackendSpring Boot 4 / Java 21
Data统一课程 JSON

把更多学校接入同一个课表体验

新增学校时，只需要按后端学校适配结构实现配置、登录会话解析与课程转换。
